# API Versioning

## Current

- Legacy routes: `/api/...`
- Versioned routes: `/api/v1/...`

## Compatibility Window

- `v1` is the stable contract.
- Legacy routes remain available during migration period and are validated by compatibility checks.

## Deprecation

- Deprecation notices are documented in `CHANGES.md`.
- Migration steps are generated by `scripts/migration_assistant.py`.

## Migration

1. Move clients from unversioned routes to `/api/v1`.
2. Run `python scripts/migration_assistant.py --strict`.
3. Validate in CI with release compatibility matrix.
